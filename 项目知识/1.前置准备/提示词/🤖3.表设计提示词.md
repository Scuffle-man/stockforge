### 高效提示词要素

| 要素 | 你的表达示例 | 作用 |
|------|--------------|------|
| **1. 明确业务场景** | "扫码枪入库、采购入库、厂直订单、改配、财务成本计算" | 让模型快速锁定领域知识 |
| **2. 给出核心约束** | "SN唯一、采购数量校验SN数量、库存预警、配件不够提示" | 约束是设计的关键，避免自由发挥偏离实际 |
| **3. 说明现有资源/技术栈** | "用 Python 和 Docker MySQL 做网页前后端分离" | 技术栈决定方案可行性，省去试探性提问 |
| **4. 罗列需管理的实体** | "采购入库表、价格表、库存表、入仓表、厂直表、备件库、京东退货、返修配件" | 即使不穷尽字段，也能让模型构造核心表关系 |
| **5. 提出合并/简化的意愿** | "入仓和厂直如果能合成一个也合成一个吧" | 允许模型进行抽象和归一化，减少冗余 |
| **6. 坦承不完美，请求优化** | "我设计的可能不好，你根据我的功能优化设计""字段有好多没给你，你凑活来" | 赋予模型重构权，激发更优方案 |
| **7. 分步推进（先表后界面）** | "前端现在先不搞，先搞表" | 控制输出范围，聚焦当前阶段 |

### 为什么这个提示词高效？

1. **信息密度高**：场景、约束、技术栈、实体、诉求一次性给出。  
2. **目标分层**：明确“先做表”，避免分散到前端。  
3. **允许模型做主**：主动请求优化，激发模型领域经验。  
4. **关键细节不遗漏**：SN唯一性、数量校验、预警、搜索、成本计算等核心逻辑均已涵盖。  
5. **可执行性强**：技术栈清晰（Py+MySQL+Docker），模型可直接输出 DDL 和环境配置。
