在本轮长达 30+ 回合的交互中，你通过不断纠正我的输出方向，实际上展示了一套**高效约束 AI 行为**的提示词方法论。下面为你提炼出 6 条核心技巧，每条都附**定义说明** + **你的原话示例**，最后给出可直接复用的万能模板。

### 🧠 技巧 1：结构强制法

**说明**：明确要求输出必须包含“并列方案”或“分层结构”（如正常流程 + 异常排查），防止 AI 堆砌单一路径。**你的原句**：

> *“你先正常流程后排查...还是需要两个部分”*
> *“全部重新设计，我需要一步步实现这个”*
> **作用**：迫使输出形成 `Part 1: 标准步骤` + `Part 2: 故障字典` 的双层架构。

---

### 🧠 技巧 2：场景重置法（上下文清空）

**说明**：用极端词语（“所有都是新的”、“从零开始”、“就当...”）强制 AI 忽略对话历史中的残留变量，基于当前时刻的纯净状态重新生成。**你的原句**：

> *“现在还有个 main，我也要删掉，所有都是新的，就当本地只有一个文件夹和里面的文件，我要所有都变成新的，你再重新输出一版”*
> **作用**：切断 AI 对前文“已经配置过 SSH/已登录”等状态的记忆，重新设计起点。

---

### 🧠 技巧 3：环境锁定法

**说明**：明确指定操作系统、终端类型、工具链（CMD/PowerShell/Git Bash），杜绝 AI 输出跨平台通用但实际无法运行的命令。**你的原句**：

> *“这改上面这块，...'Test-Path' is not recognized...重新输出一版”*
> *“我不想搞这个我想所有都在 vscode 的 Cmd 中执行”*
> **作用**：将输出从抽象的“通用 Linux/Unix 风格”强行拉回 Windows CMD 具体实现。

---

### 🧠 技巧 4：局部修正指令

**说明**：在大段输出中，只点出唯一需修改的节点，要求“只改这一处，其余照搬”，避免 AI 重写全文引发新错误。**你的原句**：

> *“你只修改这一个地，重新输出全部”*
> **作用**：精准定位错误点（如命令中的路径或参数），保留已验证的其他内容不变。

---

### 🧠 技巧 5：工具链准入限制

**说明**：明确拒绝某些工具或操作方式（如“不要打开浏览器”“不用手动创建仓库”），强制 AI 使用命令行替代方案。**你的原句**：

> *“我不想搞这个我想所有都在 vscode 的 Cmd 中执行，你重新弄。”*
> **作用**：屏蔽 `gh repo create` 或网页交互的依赖，推动纯 CLI 自动化方案。

---

## 🧩 万能提示词模板（可直接复用）

将此模板填空后发给任何 AI，即可复刻本次对话的约束效果：

---

```markdown
# 任务：将本地项目 [项目路径] 推送到 GitHub 私有仓库 [仓库名]

## 环境约束
- 操作系统：Windows
- 终端类型：CMD（命令提示符）
- 禁止使用：浏览器手动创建仓库 / PowerShell 专有命令（如 Test-Path）
- 必须使用：Git + GitHub CLI（gh）

## 输出要求
1. 按“标准操作流程” + “常见错误排查”两部分结构输出。
2. 每条命令必须单独成行，且注明预期输出。
3. 每一步后面必须附带验证方法（如 `git remote -v` 检查）。
4. 假定当前目录下没有任何 `.git` 文件夹，远程仓库也不存在，一切从零开始。

## 附加限制
- 如果某一步失败，不要继续，给出明确的回滚或重试指令。
- 所有交互（如 `gh auth login`）必须写明每一步该输入什么（例如选 SSH，输 yes，复制验证码等）。
- 最终给出一个检查清单，确保用户执行完毕后可自检。

请基于以上要求，重新生成完整操作指南。
```
