### 2. MySQL 容器部署与日常管理

#### ⚠️ 核心概念（必读）
- **容器**（Container）：MySQL 的运行实例，删除后可以重新创建。
- **数据卷**（Volume）：存放数据库文件，与容器分离，**删除容器不会丢失卷里的数据**。
- 只要挂载同一个数据卷 `mysql-data`，**之前建的表、存储的数据都会保留**。

---

#### 📦 首次部署（或希望重建容器但保留数据）
> 适用于：第一次安装，或容器损坏需要重建。

```powershell
# 1. 清理可能存在的同名旧容器（不影响数据卷）
docker rm -f mysql 2>$null

# 2. 创建并启动新容器，挂载数据卷 mysql-data
docker run -d --name mysql `
  -p 3306:3306 `
  -e MYSQL_ROOT_PASSWORD=123456 `
  -e MYSQL_ROOT_HOST=% `
  -v mysql-data:/var/lib/mysql `
  --restart unless-stopped `
  mysql:8.0 `
  --character-set-server=utf8mb4 `
  --collation-server=utf8mb4_unicode_ci `
  --default-authentication-plugin=mysql_native_password
```
> 执行后容器即会启动，之前的数据（如果卷已存在）会自动加载。

---

#### 🔁 日常启动 / 重启
> 适用于：电脑重启、容器停止后恢复。

```powershell
# 直接启动已有容器（不删除数据，不重建）
docker start mysql
```
> 如果容器已在运行，可以跳过；若想重启（如修改配置后）：
> ```powershell
> docker restart mysql
> ```

---

#### 🧹 彻底重置（清空所有数据）
> ⚠️ **会删除所有数据库和表**，仅当你想完全清空重建时使用。

```powershell
# 1. 删除容器
docker rm -f mysql

# 2. 删除数据卷（清空全部数据）
docker volume rm mysql-data

# 3. 重新创建容器（同首次部署）
docker run -d --name mysql `
  -p 3306:3306 `
  -e MYSQL_ROOT_PASSWORD=123456 `
  -e MYSQL_ROOT_HOST=% `
  -v mysql-data:/var/lib/mysql `
  --restart unless-stopped `
  mysql:8.0 `
  --character-set-server=utf8mb4 `
  --collation-server=utf8mb4_unicode_ci `
  --default-authentication-plugin=mysql_native_password
```

---

#### 📖 命令参数注释

| 命令 / 参数 | 类型 | 作用 |
|-------------|------|------|
| `docker rm -f mysql` | 清理 | 强制删除已有容器（保留卷） |
| `docker volume rm mysql-data` | 清理 | 删除数据卷（清空所有数据库） |
| `docker start mysql` | 启动 | 启动已存在的容器 |
| `docker restart mysql` | 启动 | 重启已运行的容器 |
| `-d --name mysql` | 运行参数 | 后台运行，容器命名为 `mysql` |
| `-p 3306:3306` | 运行参数 | 端口映射（本机:容器） |
| `-e MYSQL_ROOT_PASSWORD=123456` | 环境变量 | 设置 root 密码 |
| `-e MYSQL_ROOT_HOST=%` | 环境变量 | 允许 root 远程连接 |
| `-v mysql-data:/var/lib/mysql` | 数据持久化 | 挂载命名卷，数据不丢失 |
| `--restart unless-stopped` | 重启策略 | Docker 启动时容器自动启动 |
| `--character-set-server=utf8mb4` | MySQL 配置 | 支持 emoji 字符集 |
| `--collation-server=utf8mb4_unicode_ci` | MySQL 配置 | 字符排序规则 |
| `--default-authentication-plugin=mysql_native_password` | MySQL 配置 | 兼容旧版客户端认证 |

---

> 💡 **记住**：平时只需用 `docker start mysql` 启动，只有第一次或想重建容器时才用 `docker rm -f mysql` + `docker run` 那条组合命令。数据卷 `mysql-data` 是你所有数据库的“保险箱”。

#### 📖 命令注释
| 参数                                                      | 作用                                      |
| ------------------------------------------------------- | --------------------------------------- |
| `docker rm -f mysql 2>$null`                            | 删除旧容器，释放端口（不显示错误）                       |
| `-d --name mysql`                                       | 后台运行，容器命名为 `mysql`                      |
| `-p 3306:3306`                                          | 本机端口:容器端口                               |

### 4. 安装 Go 驱动并测试连接

#### 4.1 创建项目并初始化模块
m和c可以不执行，在golang终端里面执行gomod
```powershell
mkdir mysql-test
cd mysql-test
go mod init mysql-test
```

#### 4.2 获取 MySQL 驱动
在golang终端里面执行
```powershell
go get github.com/go-sql-driver/mysql
```

#### 4.3 编写测试代码
创建 `test_mysql.go`，粘贴以下内容：
```go
package main

import (
	"database/sql"
	"fmt"
	"log"

	_ "github.com/go-sql-driver/mysql"
)

func main() {
	// 连接字符串：用户名:密码@tcp(地址:端口)/数据库
	dsn := "root:123456@tcp(127.0.0.1:3306)/mysql"

	db, err := sql.Open("mysql", dsn)
	if err != nil {
		log.Fatal("❌ 打开连接失败：", err)
	}
	defer db.Close()

	// 验证连接
	err = db.Ping()
	if err != nil {
		log.Fatal("❌ 连接失败：", err)
	}

	// 获取 MySQL 版本
	var version string
	err = db.QueryRow("SELECT VERSION()").Scan(&version)
	if err != nil {
		log.Fatal("❌ 查询版本失败：", err)
	}

	fmt.Println("✅ 连接成功，MySQL 版本：", version)
}
```

#### 4.4 运行测试
```powershell
go run test_mysql.go
```
显示 **✅ 连接成功，MySQL 版本：8.0.xx** 即完成。

## 🛠️ 故障排查（仅当出错时查阅）

### ❌ 2003 (10061) 无法连接
- **原因**：容器未启动或端口映射失败
- **解决**：重新执行 **步骤 2** 的命令

### ❌ 1045 Access denied for 'root'@'172.17.0.1'
- **原因**：远程授权未生效（数据卷残留旧权限）
- **解决**：强制授权（密码 `123456`）：
  ```powershell
  docker exec -it mysql mysql -uroot -p123456 -e "UPDATE mysql.user SET host='%' WHERE user='root'; FLUSH PRIVILEGES; ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '123456'; FLUSH PRIVILEGES;"
  ```
  若仍提示密码错误，请执行「彻底重置」。

### ❌ 1045 Access denied 且密码正确
- **原因**：容器内密码与预期不一致
- **解决（彻底重置，数据清空）**：
  ```powershell
  docker rm -f mysql
  docker volume rm mysql-data
  docker run -d --name mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -e MYSQL_ROOT_HOST=% -v mysql-data:/var/lib/mysql --restart unless-stopped mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci --default-authentication-plugin=mysql_native_password
  ```
  重置后密码绝对为 `123456`，且远程权限生效。

### ❌ 1251 认证插件错误
- **原因**：客户端不支持新认证方式
- **解决**：已通过启动参数强制 `mysql_native_password`，无需额外操作。若仍有问题，执行强制授权命令（上一条）即可。

### ❌ 端口 3306 被占用
- **解决**：改用其他端口，启动时 `-p 3307:3306`，并将 Go 代码中 `3306` 改为 `3307`。

### ❌ mysql驱动无法下载
- ## 执行下面操作
你遇到的 `go get` 错误是网络连接超时，说明当前环境无法直接访问 Go 官方模块代理 `proxy.golang.org`（IP 142.250.73.145 可能是 Google 相关服务）。这在国内或受限网络中很常见。

## 解决方案：更换 Go 模块代理

使用国内可用的 Go 代理，例如 `https://goproxy.cn`（七牛云提供，速度快且稳定）。在 PowerShell 中执行以下命令：

```powershell
go env -w GOPROXY=https://goproxy.cn,direct
```

然后再重新获取依赖：

```bash
go get github.com/go-sql-driver/mysql
```

此时应该能正常下载。以后所有 `go get` 和 `go mod tidy` 都会走这个代理。

### 如果仍然失败

1. **检查 PowerShell 的执行权限**：`go env -w` 会写入 `%USERPROFILE%\go\env` 文件，一般不会有权限问题。
2. **公司网络限制**：如果你在需要代理的企业内网，还需额外设置 `HTTP_PROXY`/`HTTPS_PROXY` 环境变量，或使用 VPN。
3. **手动下载依赖包**（最后手段）：
   - 可以在能访问互联网的机器上，将 `$GOPATH/pkg/mod/cache` 下的缓存拷贝到本机对应位置，但操作较繁琐，不推荐。
