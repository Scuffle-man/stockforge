### 2. 一键部署 MySQL（自动清理旧容器）
```powershell
docker rm -f mysql 2>$null; docker run -d --name mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -e MYSQL_ROOT_HOST=% -v mysql-data:/var/lib/mysql --restart unless-stopped mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci --default-authentication-plugin=mysql_native_password
```

#### 📖 命令注释
| 参数 | 作用 |
|------|------|
| `docker rm -f mysql 2>$null` | 删除旧容器，释放端口（不显示错误） |
| `-d --name mysql` | 后台运行，容器命名为 `mysql` |
| `-p 3306:3306` | 本机端口:容器端口 |
| `-e MYSQL_ROOT_PASSWORD=123456` | root 密码设为 `123456` |
| `-e MYSQL_ROOT_HOST=%` | 允许 root 从**任意 IP** 连接（解决 172.17.0.1 拒绝） |
| `-v mysql-data:/var/lib/mysql` | 数据持久化（删除容器数据不丢） |
| `--restart unless-stopped` | Docker 启动时容器自启 |
| `mysql:8.0` | 官方镜像，本地不存在则自动下载 |
| `--character-set-server=utf8mb4` | 字符集支持 Emoji |
| `--default-authentication-plugin=mysql_native_password` | 使用旧版密码认证，兼容所有客户端 |


### 5. Python 连接测试
创建 `test_mysql.py`，粘贴以下代码：
```python
import mysql.connector

try:
    conn = mysql.connector.connect(
        host='127.0.0.1',
        port=3306,
        user='root',
        password='123456',   # 密码为 123456
        database='mysql'
    )
    print("✅ 连接成功，MySQL 版本：", conn.get_server_info())
    conn.close()
except Exception as e:
    print("❌ 连接失败：", e)
```
运行：
```powershell
python test_mysql.py
```
显示 **✅ 连接成功** 即完成。

## 🛠️ 故障排查（仅当出错时查阅）

### ❌ 2003 (10061) 无法连接
- **原因**：容器未启动或端口映射失败
- **解决**：重新执行 **步骤 2** 的命令

### ❌ 1045 Access denied for 'root'@'172.17.0.1'
- **原因**：远程授权未生效（数据卷残留旧权限）
- **解决**：强制授权（密码 `123456`）：
  ```powershell
  docker exec -it mysql mysql -uroot -p123456 -e "UPDATE mysql.user SET host='%' WHERE user='root'; FLUSH PRIVILEGES; ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '123456'; FLUSH PRIVILEGES;"
  ```
  若仍提示密码错误，请执行「彻底重置」。

### ❌ 1045 Access denied 且密码正确
- **原因**：容器内密码与预期不一致
- **解决（彻底重置，数据清空）**：
  ```powershell
  docker rm -f mysql
  docker volume rm mysql-data
  docker run -d --name mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -e MYSQL_ROOT_HOST=% -v mysql-data:/var/lib/mysql --restart unless-stopped mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci --default-authentication-plugin=mysql_native_password
  ```
  重置后密码绝对为 `123456`，且远程权限生效。

### ❌ 1251 认证插件错误
- **原因**：客户端不支持新认证方式
- **解决**：已通过启动参数强制 `mysql_native_password`，无需额外操作。若仍有问题，执行强制授权命令（上一条）即可。

### ❌ 端口 3306 被占用
- **解决**：改用其他端口，启动时 `-p 3307:3306`，后续连接使用 3307 端口。
